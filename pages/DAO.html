DAO - Data Access Object

El DAO encapsula todas las operaciones necesarias para crear, leer, actualizar o eliminar registros (CRUD) sin que el resto de la aplicación tenga que preocuparse por los detalles técnicos de SQL, conexiones o sentencias. Esto hace que el código sea más ordenado, fácil de mantener y reutilizable, ya que si cambia la base de datos o la forma en que se accede a ella, solo se modifica el DAO y no toda la aplicación.

El patrón DAO (Data Access Object) es fundamental para separar la lógica de acceso a datos de la lógica de negocio en una aplicación. Esto facilita la mantenibilidad, reutilización y pruebas del código.

Para crear y revisar correctamente un DAO en NetBeans 21, sigue estos pasos:

1. Crea o abre un proyecto Java Maven en NetBeans 21.
2. Dentro del proyecto, crea una interfaz DAO que defina los métodos para acceso a datos (como crear, leer, actualizar y eliminar).
3. Implementa la interfaz DAO en una clase concreta donde se gestione la conexión a la base de datos.
4. Asegúrate que las dependencias necesarias para acceder a la base de datos estén declaradas en el pom.xml.
5. Usa NetBeans para detectar errores en tu código DAO y realiza las correcciones necesarias.
6. Refresca y limpia el proyecto para aplicar cambios y compilar sin errores.
7. Prueba el DAO integrándolo con la lógica de negocio o mediante pruebas unitarias.
8. Finalmente, verifica que las operaciones de acceso a datos funcionan correctamente y sin excepciones.
