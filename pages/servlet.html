Servlet CRUD - Explicación de métodos

Un servlet es un componente de Java que se ejecuta en un servidor web o de aplicaciones y que permite procesar solicitudes y generar respuestas de manera dinámica. Funciona como un intermediario entre el cliente (por ejemplo, un navegador) y la lógica de la aplicación, recibiendo peticiones HTTP, ejecutando el código necesario y devolviendo una respuesta, que normalmente es una página HTML o datos en otros formatos como JSON o XML. Los servlets son la base de muchas aplicaciones web en Java, ya que facilitan la creación de contenido dinámico y la interacción con bases de datos y otros recursos del servidor.

El servlet CategoriaServlet maneja las operaciones CRUD (Crear, Leer, Actualizar, Eliminar) para la entidad Categorías. La lógica se divide principalmente en los métodos doGet() y doPost(), según el tipo de operación y la acción recibida.

doGet(HttpServletRequest request, HttpServletResponse response)

Este método procesa las solicitudes HTTP GET y responde principalmente a operaciones de lectura y navegación:

  • Listar: Recupera todas las categorías usando el DAO y las pasa como atributo a la página Categorias.jsp para mostrar la lista.
  • Editar: Obtiene el ID de la categoría a editar, busca la categoría con el DAO, la pasa a editarCategoria.jsp para mostrar el formulario con datos actuales.
  • Eliminar: Recibe el ID, elimina la categoría a través del DAO y redirige a la lista para mostrar los cambios.
  • Default: Si no se especifica acción, redirige a la lista por defecto.

doPost(HttpServletRequest request, HttpServletResponse response)

Este método procesa las solicitudes HTTP POST, usadas para operaciones que modifican datos (crear o actualizar):

  • Insertar: Crea un nuevo objeto Categoría con los datos recibidos del formulario, lo guarda mediante el DAO y redirige a la lista.
  • Actualizar: Recupera la categoría existente por ID, actualiza sus datos con los valores recibidos y guarda los cambios con el DAO, luego redirige a la lista.
  • Default: Redirige a la lista si no se especifica la acción o no coincide con las anteriores.

Este enfoque centraliza la gestión de acciones mediante un parámetro accion que determina qué operación se realiza, manteniendo el servlet organizado y evitando múltiples servlets para cada operación.
